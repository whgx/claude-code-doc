## 1. 什么是 Claude Code？

### 1.1 简单理解

**Claude Code** 就是一个**会写代码的 AI 助手**，它运行在你的终端里。

打个比方：
- 你就像和一个经验丰富的程序员聊天
- 告诉它你想做什么
- 它会帮你**写代码**、**改代码**、**找 bug**
- 而且它可以直接操作你的项目文件

### 1.2 它能做什么？

| 你需要做的 | Claude Code 会做的 |
|----------|------------------|
| 想添加一个登录功能 | 写出完整的登录代码 |
| 程序报错了 | 找出问题并修复 |
| 看不懂某个文件 | 解释代码逻辑 |
| 想优化代码 | 重写成更优雅的版本 |
| 想写测试 | 自动生成测试代码 |
| 想改代码风格 | 批量修改格式 |

### 1.3 和普通 AI 聊天有什么不同？

普通 AI 聊天（如 ChatGPT）：
```
你：帮我写一个登录页面
AI：给你一段代码
你：复制代码，粘贴到文件，保存，运行
你：有问题，再问 AI
```

Claude Code：
```
你：帮我写一个登录页面
Claude：[自动]创建文件 → 写入代码 → 运行测试 → 告诉你完成了
```

**核心优势：**
- ✅ 直接操作你的项目文件
- ✅ 能运行命令
- ✅ 知道整个项目结构
- ✅ 不需要你复制粘贴

## 2. 安装指南

### 2.1 系统要求

| 系统 | 要求 |
|------|------|
| macOS | 10.15 及以上 |
| Windows | Windows 10 或 11 |
| Linux | Ubuntu 20.04+ / Debian 10+ |
| 内存 | 最低 4GB（推荐 8GB+） |
| 账户 | Claude 订阅（Pro、Max、Teams） |

### 2.2 安装步骤

#### macOS 用户（最简单）

```bash
# 打开终端，复制下面这一行，回车
curl -fsSL https://claude.ai/install.sh | bash
```

安装完成后，验证：
```bash
claude --version
# 应该显示：claude-code x.x.x
```

#### Windows 用户

**方法一：PowerShell（推荐）**
```powershell
# 打开 PowerShell，复制下面这行
irm https://claude.ai/install.ps1 | iex
```

**方法二：CMD**
```cmd
# 打开 CMD，复制下面这行
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

**方法三：WinGet**
```cmd
winget install Anthropic.ClaudeCode
```

#### Linux 用户

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

### 2.3 首次登录

安装完成后，第一次使用需要登录：

```bash
# 进入你的项目目录
cd your-project

# 启动 Claude Code
claude
```

会看到类似这样的提示：
```
✨ Welcome to Claude Code!

Please authenticate to continue:

1. Press Enter to open your browser
2. Complete the sign-in process
3. Return here to continue

Press Enter to open browser...
```

按照提示操作：
1. 按回车键
2. 浏览器会自动打开登录页面
3. 用你的 Claude 账号登录
4. 登录成功后，回到终端继续

## 3. 第一次使用

### 3.1 进入你的项目

```bash
# 假设你有一个 Web 项目
cd /path/to/your-project

# 启动 Claude Code
claude
```

进入后会看到类似这样的界面：
```
╭─────────────────────────────────────────────────────╮
│  Claude Code v2.1.2                              │
│  Connected to model: Claude Sonnet 4.5            │
│  Session: /Users/xxx/my-project                   │
╰─────────────────────────────────────────────────────╯

> _
```

`> _` 就是输入提示符，你在这里输入命令。

### 3.2 简单演示：创建一个 Hello World

让我们用 Claude Code 创建一个简单的 HTML 页面：

```bash
> 创建一个名为 hello.html 的文件，内容是一个简单的登录页面
```

Claude 会自动：
1. 创建文件
2. 写入 HTML 代码
3. 告诉你完成了

看到类似这样的输出：
```
✓ Created hello.html

文件内容：
<!DOCTYPE html>
<html>
<head>
    <title>登录页面</title>
    ...
</head>
<body>
    ...
</body>
</html>
```

### 3.3 查看文件

```bash
> 读取 hello.html 文件的内容
```

Claude 会显示文件内容。

### 3.4 修改文件

```bash
> 把 hello.html 的标题改成"欢迎登录"
```

Claude 会自动修改文件。

### 3.5 运行项目

```bash
> 用浏览器打开 hello.html
```

或者：
```bash
# 假设是 Node.js 项目
> 运行 npm start
```

Claude 会执行命令并显示结果。

## 4. 基础功能演示

### 4.1 问问题

```bash
# 了解项目结构
> 这个项目是用什么框架写的？

# 解释代码
> 解释 src/auth.js 文件中的登录逻辑

# 查找文件
> 找出所有包含 "TODO" 注释的文件

# 统计信息
> 这个项目有多少行代码？
```

### 4.2 写代码

```bash
# 添加新功能
> 添加一个用户注册功能，包括输入用户名、密码、确认密码

# 生成测试
> 为 User 类编写单元测试

# 添加注释
> 给 src/utils.js 添加详细的注释
```

### 4.3 改代码

```bash
# 重构
> 把 index.js 里的 callback 函数改成 Promise

# 优化
> 优化这段代码的性能：[粘贴代码]

# 格式化
> 把项目里所有的文件按照 Prettier 规则格式化
```

### 4.4 找 Bug

```bash
# 粘贴错误信息
> 程序运行时报错：TypeError: Cannot read property 'xxx' of undefined

# 描述问题
> 登录功能不工作，用户输入正确的密码也无法登录
```

Claude 会：
1. 查看相关代码
2. 分析问题
3. 提出修复方案
4. 自动修改代码

## 6. 新手常见问题

### 6.1 安装问题

**Q: 安装时提示 "command not found"**

A: 可能原因和解决方法：
```bash
# 1. 检查是否安装成功
which claude

# 2. 如果没有，添加到 PATH
export PATH="$HOME/.claude/bin:$PATH"

# 3. 永久添加（添加到 ~/.zshrc 或 ~/.bashrc）
echo 'export PATH="$HOME/.claude/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**Q: 登录时浏览器没有打开**

A: 手动打开浏览器访问终端显示的 URL。

### 6.2 使用问题

**Q: Claude 修改了错误的文件怎么办？**

A: 撤销修改：
```bash
# Git 项目
git checkout -- <文件名>

# 或者让 Claude 撤销
> 撤销对 xxx.js 的修改
```

**Q: 如何查看 Claude 做了哪些修改？**

A:
```bash
# Git 项目
git diff

# 或者问 Claude
> 显示你刚才做了哪些修改
```

**Q: Claude 可以操作我的哪些文件？**

A: 默认情况下，Claude 可以读写项目目录下的所有文件。

**限制权限**：
```bash
# 创建配置文件
cat > .claude/settings.json << EOF
{
  "permissions": {
    "deny": [
      "Bash(rm -rf:*)",
      "Read(.env*)"
    ]
  }
}
EOF
```

### 6.3 成本问题

**Q: 使用 Claude Code 需要花钱吗？**

A: 需要，但价格很合理：
- **Haiku**: 输入 $1/百万 token，输出 $5/百万 token
- **Sonnet**: 输入 $3/百万 token，输出 $15/百万 token
- **Opus**: 输入 $5/百万 token，输出 $25/百万 token

**Q: 如何查看使用量？**

A:
```bash
> /cost
```

会显示类似这样的信息：
```
会话统计：
- 模型：Claude Sonnet 4.5
- 输入 Token：12,345
- 输出 Token：6,789
- 预估费用：$0.15
```

**Q: 如何节省成本？**

A:
1. 使用 `/compact` 压缩对话历史
2. 简单任务使用 Haiku 模型
3. 清晰地描述需求，减少往返对话
4. 使用子代理处理简单任务

### 6.4 技术问题

**Q: Claude Code 支持哪些编程语言？**

A: 几乎所有主流语言：
- JavaScript / TypeScript
- Python
- Java
- Go
- Rust
- C/C++
- PHP
- Ruby
- 等等...

**Q: 可以在大型项目中使用吗？**

A: 可以，Claude 会智能选择需要阅读的文件，不会一次性加载整个项目。

**Q: 支持团队协作吗？**

A: 支持！详见《进阶使用手册》的团队协作章节。

|------|----------|----------|
